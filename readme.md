🗄️ Database Architecture (Already initialized via setup.php)

The setup.php file automatically generates the database and the following tables. Assume these exist in your development environment:

school_settings: Stores school_name, logo_path, and gemini_api_keys (comma-separated for quota rotation).

admins: Admin credentials (username, password_hash).

class_seats: Tracks total_seats per class (ECD to Class 9, Class 11).

faculties: +2 Streams (Science, Management) and requires_entrance boolean.

faculty_subjects: Optional subjects mapped to faculty_id.

entrance_schedules: Exam slots with total_capacity to prevent overbooking.

admission_inquiries: The main monolithic table storing all student data, PP photo paths, cascading address data, parent info, and entrance roll numbers.

Step 4: Document Uploads (PP Photo, Marksheet) & Declaration.

Feature: Add an "AI Auto-fill" button next to the Marksheet upload that triggers the Gemini OCR endpoint via Fetch API.

⚡ CRITICAL FRONTEND JS LOGIC (form-wizard.js):
Implement these strict academic rules dynamically in the UI:

Class 9 Optional Subjects Rule:

If "Class 9" is selected, reveal Optional I and Optional II dropdowns.

Rule: If Optional I is selected as "Economics", Optional II MUST be auto-selected to "Account" and disabled (read-only). If Optional I is "Math", the user can choose either "Account" or "Computer" for Optional II.

Class 11 (+2) Logic:

If "Class 11" is selected, reveal Faculty dropdown.

Fetch subjects mapped to faculties. (e.g., Science -> Computer or Biology; Management -> Computer, Business Studies, or Hotel Management).

If the chosen Faculty has requires_entrance = 1 (like Science), reveal the Entrance Slot booking section (fetching available slots from entrance_schedules).

Date Masking:

For the BS Date input (dob_bs), use JS to auto-insert hyphens as they type (format: YYYY-MM-DD). E.g., typing "208301" becomes "2083-01".

🧠 Phase 4: Backend Processing (process_form.php & ocr_endpoint.php)

ocr_endpoint.php: Receives an uploaded image, sends it to the Gemini API vision model via cURL, and returns JSON containing extracted student_name, dob, and gpa.

process_form.php:

Securely handle file uploads (move to /assets/uploads/documents/). Ensure unique file names.

Check capacity constraints for entrance slots.

Insert the record into admission_inquiries.

Redirect to admit_card.php (if applicable) or a success page.

📊 Phase 5: Admin Dashboard (login.php & dashboard.php)

Secure session-based login.

Mint-themed dashboard to view applications, change statuses, and export data to CSV.

⚠️ Strict Agent Rules

Mobile First: Ensure all Tailwind form inputs have adequate padding (e.g., p-3) for touch targets on mobile devices.
